# Proposal: Rebrand FoloUp → Robust Devs Hiring

## What

A text-only brand swap that replaces every user-visible and configuration-level reference to "FoloUp" / "folo-up" with the new product identity "Robust Devs Hiring", scoped strictly to in-repo files. No functional logic changes. No database migrations. No cloud-resource renames.

## Why

The product is being rebranded. The old name ("FoloUp") is tied to the original open-source framing and the `folo-up.co` domain. The new identity ("Robust Devs Hiring") reflects the tool's purpose (internal hiring workflow) and the agency brand (`robustagency.co`). Keeping stale brand strings in metadata, OG tags, UI text, contact emails, and documentation creates inconsistency and potential embarrassment when demoing or handing off to a client.

## Scope

| Category | Items |
|---|---|
| `package.json` `name` field | `foloup-app` → `robust-devs-hiring-app` |
| `<head>` metadata (title, OG title, OG siteName) | `FoloUp` → `Robust Devs Hiring` |
| OG image asset | `public/FoloUp.png` → `public/robust-devs-hiring.png` (git mv) |
| JSX brand spans (7 locations) | Rewrite using chosen split (see design.md) |
| Plain-text UI strings | 1 description string in interviewers page |
| Hardcoded fallback URLs (`https://folo-up.co/`) | 4 locations → `https://robustagency.co` |
| `.env.example` marketing URL | `https://folo-up.co` → `https://robustagency.co` |
| Contact email | `founders@folo-up.co` → `hi@robustagency.co` |
| Code comments mentioning old brand | Swap brand name text, keep comment structure |
| `docker-compose.yml` service name | `foloup` → `app` |
| `README.md` | Full rewrite as internal-tool docs |
| `CONTRIBUTING.md` | Brief internal-contribution guide (decision documented in design.md) |

## Assumptions

1. The `text-brand-bold` Tailwind class and `--ds-brand-bold` CSS variable are generic styling tokens and will not be renamed.
2. The Supabase project name, Vercel project name, GitHub repo name, and local directory name are renamed separately by the operator in their respective dashboards/filesystem — not in this change.
3. `package-lock.json` will be regenerated automatically when the operator runs `npm install` after `package.json` is updated; it is not hand-edited.
4. The OG image file is renamed only (not redesigned); `public/FoloUp.png` is the same image rehosted under the new filename.
5. All files under `openspec/changes/archive/` are treated as immutable historical records and are not touched.

## Non-goals

- Renaming the local directory `/Users/tayyab/Projects/foloup`
- Renaming Supabase project, Vercel project, or GitHub repo
- Any database schema or column renames
- Logo redesign or new brand identity assets
- Renaming the `text-brand-bold` Tailwind class
- Renaming the `--ds-brand-bold` CSS variable
- Editing `package-lock.json` directly (regenerated by `npm install`)
- Modifying anything under `openspec/changes/archive/`
