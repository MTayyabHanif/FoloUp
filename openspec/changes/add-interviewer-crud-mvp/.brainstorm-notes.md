## Decisions

### 1. Multi-tenancy scope
**Decision:** Globally shared (no scoping change).
**Implication:** No `org_id` / `user_id` column added. `getAllInterviewers` continues to return all non-deleted rows. Document in proposal.md as an explicit non-goal: this MVP is for the single-org case; multi-tenancy is a follow-up.

### 2. Prompt storage
**Decision:** Add `prompt TEXT NOT NULL` to `interviewer`. Migrate all three existing rows.
**Implication:**
- Schema migration adds the column (probably with a temporary default of empty string, then backfill, then `ALTER COLUMN SET NOT NULL`).
- A one-time backfill writes the three known prompts to the three known rows by name (Lisa/Bob use `RETELL_AGENT_GENERAL_PROMPT`, Robust Bot uses `RETELL_AGENT_ROBUST_BOT_PROMPT`).
- `RETELL_AGENT_GENERAL_PROMPT` and `RETELL_AGENT_ROBUST_BOT_PROMPT` constants can stay in `constants.ts` as "seed source" with a deprecation comment, OR be removed if no other code references them. propose / plan-eng-review will decide.
- The `create-interviewer` route is now SEED-ONLY: it should be marked deprecated or be removed in favor of the new `POST /api/interviewers` flow.

### 3. Delete semantics
**Decision:** Soft-delete only via `deleted_at TIMESTAMPTZ NULL`.
**Implication:**
- Schema migration adds the column.
- `getAllInterviewers` filters `WHERE deleted_at IS NULL`.
- A new `getInterviewerById` used by `register-call` must NOT filter on `deleted_at` (so soft-deleted interviewers can still serve in-flight interviews).
- The DELETE endpoint sets `deleted_at = NOW()`, does NOT call Retell delete APIs.
- Document Retell-side leak as a known cost in design.md (acceptable for MVP).

### 4. Voice picker
**Decision:** Curated dropdown.
**Implication:**
- Add a `VOICE_OPTIONS` constant (4-6 entries) to `src/lib/constants.ts` shaped like `{ id: "11labs-Brian", label: "Brian (warm, neutral male)" }`.
- Operator-supplied entries: propose step should ask operator for the 4-6 voice IDs and labels if not already known; otherwise default to a starter set that includes Chloe and Brian (already in use).
- Form renders a `<Select>` of these. No live Retell catalog fetch.

### 5. Prompt editor UX
**Decision:** Textarea + locked footer.
**Implication:**
- The locked footer text lives in a single constant (e.g., `PROMPT_FOOTER_TEMPLATE` in `constants.ts`) and is appended at submit time. Footer contains the four `{{name}}`, `{{mins}}`, `{{objective}}`, `{{questions}}` placeholders.
- Form UI shows the footer below the textarea as visually distinct, non-editable text. (Implementation detail for design review.)
- Server validates that the locked footer is present in any incoming custom prompt (defensive â€” protects against API-direct callers who skip the UI).

## Non-goals for this MVP

- Multi-tenancy / org scoping
- Per-interviewer voice or prompt EDITING (delete + recreate is the v1 mutation)
- Live Retell voice catalog
- Audio preview in the create form
- Trait sliders affecting agent behavior (they remain decorative metadata)
- Retell resource cleanup on delete (LLM + agent leak is accepted)
- Versioned prompts / prompt history
